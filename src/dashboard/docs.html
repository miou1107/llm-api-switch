LLM API Switch

免費 LLM API 聚合代理 — OpenAI 相容介面

Base URL

https://kkvin.com/llm-switch/v1

認證

所有 /v1/* 端點需要 Bearer token：

Authorization: Bearer sk-你的API金鑰

API Key 由管理員從 Dashboard 產生。

端點

POST /v1/chat/completions

聊天補全。完全相容 OpenAI Chat Completions API。

Request Body

{
  "model": "auto",
  "messages": [
    {"role": "user", "content": "你好"}
  ],
  "temperature": 0.7,
  "max_tokens": 1000,
  "stream": false
}

model 參數

auto自動選擇評分最高、額度充足的模型（推薦）
llama-3.3-70bLlama 3.3 70B（Groq / OpenRouter / NVIDIA / Together）
llama-3.1-8bLlama 3.1 8B（Groq / Cerebras / NVIDIA / SambaNova）
deepseek-r1DeepSeek R1 推理模型（OpenRouter / DeepSeek）
deepseek-v3DeepSeek V3 對話模型（OpenRouter / DeepSeek）
mistral-smallMistral Small（Mistral / OpenRouter）
gemini-2.0-flashGoogle Gemini 2.0 Flash
qwen-3-235bQwen 3 235B（Cerebras）
gpt-4oGPT-4o（GitHub Models）

也可以直接使用 provider 的原始 model ID，例如 deepseek/deepseek-r1:free

GET /v1/models

列出所有可用模型。

GET /health

健康檢查（不需認證）。回傳 {"status": "ok"}

使用範例

Python (OpenAI SDK)

from openai import OpenAI

client = OpenAI(
    base_url="https://kkvin.com/llm-switch/v1",
    api_key="sk-你的API金鑰"
)

response = client.chat.completions.create(
    model="auto",
    messages=[{"role": "user", "content": "你好"}]
)

print(response.choices[0].message.content)

curl

curl https://kkvin.com/llm-switch/v1/chat/completions \
  -H "Authorization: Bearer sk-你的API金鑰" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "auto",
    "messages": [{"role": "user", "content": "你好"}]
  }'

JavaScript (fetch)

const response = await fetch("https://kkvin.com/llm-switch/v1/chat/completions", {
  method: "POST",
  headers: {
    "Authorization": "Bearer sk-你的API金鑰",
    "Content-Type": "application/json"
  },
  body: JSON.stringify({
    model: "auto",
    messages: [{ role: "user", content: "你好" }]
  })
});
const data = await response.json();
console.log(data.choices[0].message.content);

Node.js (OpenAI SDK)

import OpenAI from "openai";

const client = new OpenAI({
  baseURL: "https://kkvin.com/llm-switch/v1",
  apiKey: "sk-你的API金鑰"
});

const completion = await client.chat.completions.create({
  model: "auto",
  messages: [{ role: "user", content: "你好" }]
});

console.log(completion.choices[0].message.content);

特性

  • model: "auto" — 自動選擇評分最高、額度充足的模型
  • 自動 fallback — 如果選中的 provider 失敗，自動切換到下一個（最多 3 次）
  • 額度感知 — 額度快用完的 provider 會被自動跳過
  • 完全相容 OpenAI API — 任何支援 OpenAI SDK 的工具都能直接使用

錯誤碼

401無效或缺少 API Key
400無效的請求（找不到 model 等）
502所有 provider 都失敗
500內部錯誤
